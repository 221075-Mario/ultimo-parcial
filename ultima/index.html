Formularios

Los formularios en HTML se definen usando la siguiente etiqueta < form>. Y sus atributos importantes son < input>: Componente clave de los formularios para que los usuarios ingresen datos.

text: Campo de texto.

email: Campo para direcciones de correo electrónico.
password: Campo para contraseñas (oculta los caracteres).
checkbox y radio: Selección múltiple y única, respectivamente.
number, range: Para valores numéricos y rangos.
date, time, datetime-local: Campos de fecha y hora.

Botones de envío y restablecimiento:
< button type="submit">: Envía los datos del formulario.
< button type="reset ">: Restaura el formulario a su estado inicial.
Diseño y accesibilidad:
Usar label con for mejora la accesibilidad.

Tablas

Ideas principales


Tablas

Las tablas en HTML son estructuras que organiza la informacion en filas y columnas, son utiles para presentar datos de una manera mas estructuradas.
Elementos de una tabla:

< table>: Define la tabla en sí.
< thead>: Define la cabecera de la tabla, ideal para títulos de columna.
< tbody>: Contiene el contenido principal de la tabla.
< tfoot>: Pie de la tabla, útil para resúmenes o totales.
< tr>: Representa una fila de la tabla.
< th>: Celda de encabezado (en negrita por defecto).
< td>: Celda de datos.

Atributos y estilos comunes:
colspan: Permite que una celda ocupe varias columnas. rowspan: Permite que una celda abarque varias filas.

Especificidad

La especificidad en HTML se refiere a cómo los navegadores deciden qué reglas de CSS aplicar cuando hay varios estilos que podrían aplicarse a un mismo elemento. Esto es crucial para entender como los navegadores resuelven los conflictos de las reglas.
¿Cómo se calcula?
La especificidad solo se aplica cuando el mismo elemento es el objetivo de varias declaraciones.
◦ Cuando varias declaraciones tienen la misma especificidad, la última declaración encontrada en CSS se aplica al elemento.

Reglas de especificidad
Igual especificidad: la última regla cuenta: si la misma regla se escribe dos veces en la hoja de estilo externa, entonces la regla inferior en la hoja de estilo está más cerca del elemento que se va a diseñar y, por lo tanto, se aplicará: siempre se aplica la última regla. tiene menú contextual

Resumen

La especificidad en CSS determina qué reglas se aplican a un elemento cuando hay varios estilos que podrían coincidir. Su prioridad se calcula según el tipo de selector utilizado:
ID selectors: Tienen la mayor especificidad. Class selectors y pseudo-clases: Tienen menor especificidad que los ID.
Element selectors y pseudo-elementos: Tienen la menor especificidad.
!important: Aumenta la prioridad de una regla, pero debe usarse con cuidado.

After y Before

Ideas principales


After y Before

En HTML, ::before y ::after son pseudo-elementos de CSS que permiten insertar contenido antes o después del contenido de un elemento sin modificar el HTML. Son útiles para añadir decoraciones, iconos, o texto sin alterar la estructura del documento.
Before
El pseudo-elemento ::before inserta contenido antes del contenido real de un elemento.
After
El pseudo-elemento ::after inserta contenido después del contenido real de un elemento.
Ambos pseudo-elementos requieren la propiedad content para funcionar.

    RGB Y RGBA:
    El formato RGB usa valores numéricos para definir los colores, especificando la cantidad de rojo, verde y azul (de 0 a 255). El formato RGBA es una extensión del formato RGB, añadiendo un valor de opacidad.
    HSL Y HSLA:
    El formato HSL describe el color usando matiz (hue), saturación (saturation) y luminosidad.Al igual que RGBA, HSLA extiende el formato HSL agregando un valor alfa para la transparencia. (lightness).
